<h2>代码与思路总结与整理:</h2>
<ol>
<li>大家有空时上传自己在群里分享的代码并把自己所有代码的思路都总结到一个文档中，<b>到自己的分支(sun,jiang,lin,wu,chai)，我会合并分支到master</b>，并且汇总大家的思路</li>
<li>代码命名格式:为了能快速找到可用的代码，leetcode题，用L打头，L题号中文题名 例如L70爬楼梯.cpp，笔试题用B打头，B笔试日期_公司_题名 例如 B8.12_头条_区间合并.cpp
如果是面试算法题，用M打头，M面试日期_公司_题名 例如 M8.20_头条_区间合并.cpp，如果是其它途径的题，直接写中文题名.cpp 如区间合并.cpp</li>
<li>思路命名格式:自己所有题的题干和思路（leetcode题无需题干，笔试面试题需要题干）都总结在一个word文档里，命名为:姓名首字母_思路.docx,例如 s_思路.docx 
使用题名做每段标题，并在题的旁边注上本题类型，方便我汇总分类，例如动态规划，字符串，并查集等
当然也可以采取一级标题，二级标题的形式</li>
<li>虽然大家在git上总结，但仍需在群里分享，便于零碎时间思考</li>
</ol>
